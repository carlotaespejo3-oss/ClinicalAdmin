PART 2 — DOCUMENT AND FORM DETECTION
When classifying any email, scan for indicators that a document needs to be completed. These include:

NDIS report or letter
School support letter
Medical certificate
Referral letter
Court or medico-legal report
Insurance form
Psychologist or allied health report request
Any attachment described as a form
Any request that says "please complete", "please provide a letter", "please fill in", "we need a report"

If any of these are detected:

Flag the email with a document badge: "📄 Document required"
Add 20 minutes to the email's time estimate to account for the document itself
Automatically create a linked task. The task should contain:

Title: "[Document type] — [patient name if detectable] — requested by [sender name]"
Category: CLINICAL or PROFESSIONAL depending on who sent it
Priority: inherited from the email's final priority
Due date: extracted from the email if mentioned. If not mentioned, default to 14 days from today
Time estimate: 20 minutes
Linked email ID: the ID of the email that generated this task
Status: incomplete


Show the linked task immediately below the email in the worklist — visually connected with a line or indent so it is clear they belong together
Show a note on the email: "A task has been created for this document. It will complete automatically when you mark this email as done."


PART 3 — LINKED EMAIL-TASK LOGIC
Tasks and emails that are linked should behave as follows:
When the clinician marks the email as done:

Check if there is a linked task
If the task is still incomplete, show a prompt: "You've marked this email as done. Have you completed the document too? [Yes — complete task] [No — keep task open]"
If they click Yes: mark the linked task as complete automatically
If they click No: keep the task open and move it to the task list with a note: "Document still needed — email already replied to"

When the clinician marks the linked task as done:

Check if the linked email is still open
If it is: show a prompt: "You've completed the document. Do you also want to mark the original email as done? [Yes] [No]"

When the clinician sends a reply to the email that says the document is in the file or has been completed:

Detect language like "please find attached", "I have completed", "the letter is in the file", "I have sent the report"
Automatically prompt: "It looks like you may have completed this document. Do you want to mark the linked task as done? [Yes] [No]"


PART 4 — DAILY PLAN DISPLAY
In the home screen daily plan and the detailed today view, when an email has a linked task:

Show the email and its linked task one after the other — never separated
The email comes first, the task comes second, indented slightly below it
Show a small link icon or connector between them so the clinician can see they belong together
The combined time estimate for the pair is shown as one total: "Email + document: approx X min total"
If the task has been completed but the email is still open, show: "Document done — reply still needed"
If the email has been replied to but the task is still open, show: "Email replied to — document still needed"
