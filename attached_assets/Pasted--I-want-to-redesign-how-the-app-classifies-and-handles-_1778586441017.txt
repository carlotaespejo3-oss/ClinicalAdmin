

I want to redesign how the app classifies and handles emails. Please read the current classification system and replace it completely with the following. This is the full spec — implement it exactly, category by category.

PART 1 — TRIAGE PRIORITY
Every email gets one of these four priority levels:
URGENT

Medication side effect query
Mental health or safety concern
Behavioural escalation
Physical symptoms
Any safeguarding concern

MEDIUM

Medication dose adjustment request
Blood test or result query
Complaint or dissatisfaction
Communication from allied health, GP, school, or other doctors
Legal or medico-legal correspondence

LOW

Prescription request
Lost or misplaced medication request
School letter request
NDIS, report, or form request
Appointment request
Billing or admin question
Triage requests
General admin

UNCLEAR

Does not fit any category above
Always flag immediately for clinician review before anything else
Never auto-draft for unclear emails
If unreviewed for more than 48 hours, escalate to urgent visibility and show: "This unclassified email has been waiting X days for your review"


PART 2 — EMAIL CATEGORIES
Every email gets one of these category labels:
SAFEGUARDING
Any mention of self-harm, suicidal ideation, abuse, neglect, risk to a child or vulnerable person, or a family expressing acute distress about safety. This takes priority over all other categories — if there is any doubt, classify as SAFEGUARDING.
URGENT_CLINICAL
Severe symptoms, severe behavioural escalation, family appears distressed or in crisis — but no safeguarding concern identified.
CLINICAL
Clinical questions about symptoms, adjusting or checking medication doses, script renewals, dose checks, simple clinical advice, routine clinical decisions.
PROFESSIONAL
Emails from colleagues waiting on the clinician — psychology, GP, school mental health lead, allied health, other doctors. Detect sub-type:

Waiting on clinical input → draft a clinical reply
Waiting on a document (report, letter, referral) → prompt "Do you want to add a task to write this?" and draft a holding reply saying it is in progress
Meeting or coordination → treat like ADMIN

ADMIN
Bookings, room changes, rota changes, forms, scheduling.
LEGAL
Medico-legal correspondence, complaints with legal implications, court-related.
NONE
No action required — newsletters, bulletins, FYI threads, cc'd threads, marketing.
CPD
Upcoming meetings, conferences, courses, or other continuing professional development activities. Extract any registration deadline mentioned and use it as the task due date.
DONE
DONE is not an AI classification. It is a manual status the clinician applies by clicking "Mark as done" on any email. Remove DONE from the AI classification list entirely. Once marked done, the email moves to an archive view.
UNCLEAR
Does not fit any category. Flag for human review. Show a prominent yellow warning: "This email needs your review before we can classify it." Allow the clinician to manually select the correct category. Once selected, apply the behaviour for that category.

PART 4 — GENERAL RULES FOR ALL DRAFTS

All drafts appear automatically when an email is opened — the clinician should never have to click a button to generate a draft
Every draft is clearly labelled "Draft — review before sending"
Drafts containing clinical content are additionally labelled "This draft contains clinical content — review carefully"
The clinician can edit any draft before sending
The clinician can completely replace a draft with their own text
Never send anything automatically — the clinician always approves and sends manually
Drafts should match the clinician's writing style profile if one has been built in the app
Tone guidelines:

For families: warm, clear, jargon-free, human
For colleagues: professional, collegial, direct
For admin: brief and practical
For legal: neutral, factual, careful — only if clinician requests
